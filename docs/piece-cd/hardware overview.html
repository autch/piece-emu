■■■■　ハードウェア概要　■■■■　　Version 1.00　　Last Update : 07/11/2001
【ハードウェアの構成】 
　P/ECEのハードウェアは大きく分けて以下のような構成になっています。

・CPU (EPSON S1C33209 24MHz)
　CPUとして、24MHzで動作するEPSONのS1C33209マイコンを使用しています。

・表示部 (LCD) 
　表示部として128*88ドットのFSTN液晶を搭載しています。白黒4階調の表現が可能です。

・サウンド (PWM音源) 
　CPU内蔵のPWM出力を使い、ソフトウェア処理により複数の自然音の再生の能力を有しています。  
　出力は、内蔵スピーカーまたは、ヘッドホン端子が使用可能です。

・メモリ
　メモリは、メインRAMとしてS-RAM256KB、システム及びファイル用としてFLASHメモリを512KB搭載
　しています。
　いずれも、高速アクセスの可能なメモリを使用しています。

・インターフェース/ペリフィラル（機器）
　4方向パッドと、START/SELECT、A、Bの4つのボタンが使えます。 
　また、背面にはリセットスイッチがあります。 

　USB …PC本体と12Mb/sの高速な転送が可能なUSBインターフェースです。 
　赤外線インターフェース …他のP/ECE等との通信に使える、赤外線インターフェースです。
【ＣＰＵ】 
　P/ECEには、EPSON製のS1C33209というCPUが搭載されています。 
　主な特徴は、以下のとおりです。 

　　24MHzクロックで動作 
　　8Kバイトの高速内蔵メモリ 
　　DMAコントローラ  
　　高速DMA4ch. 
　　インテリジェントDMA128ch. 
　　タイマ 
　　8ビットタイマ6ch. 
　　16ビットタイマ6ch. 
　　計時タイマ（アラーム機能付）
　　シリアルインターフェース
　　シリアル入出力4ch.
　　A/D変換器
　　10ビット8ch.
　　汎用入出力ポート（周辺回路の入出力と兼用）

　　入出力兼用ポート29ビット
　　高速積和演算機能
   
【8Kバイトの高速内蔵メモリ】
　・S1C33209は高速内蔵メモリを8KB持っています。このメモリを利用することにより、時間がかかる
　　外部メモリへのアクセスを減らすことが出来ます。内蔵メモリのデバイスサイズは32ビットで、
　　バイト、ハーフワード、ワードデータが1サイクルで読み出し/書き込み可能です。
　・8Kバイトのうち、いくらかはシステムで予約されています。現在のシステムで利用できるRAMは
　　別途システム関連のドキュメントを参照してください。

【DMA】  
　・DMAは大量のデータをある場所から別の場所に一気に転送するもので、DMAがデータを転送している
　　間、CPUの計算機能は並行して動くことができます。DMAを利用することにより、転送が終了するま
　　でCPUの処理が中断することがありません。
　・S1C33209には高速DMAが4、インテリジェントDMAが128ありますが、それぞれのチャネルは目的
　　ごとに異なります。  
　・高速DMAの４つのうち２つはシステムが使用するため、現時点でユーザレベルで使用できるのは
　　２つぶんです。
　・システムで予約しているDMAの１つはLCD転送に使用されます。 
　・システムで予約しているもうひとつのDMAはサウンドの転送に使用されます。
　・高速DMA及びインテリジェントDMAのシステムでの利用方法は、将来的に変更される可能性があり
　　ます。ユーザーは現システムでのみという条件付でシステムで使用していないDMAを利用すること
　　が可能です。

【タイマ】  
　・S1C33209は内部に独立して動くタイマを内蔵しています。タイマは割り込み、外部への信号
　　出力、CPUへの割り込みなどいろいろな使用方法があります。
　・S1C33209には8ビットタイマが6チャネル、16ビットタイマが6チャネル、計時タイマが1チャネル
　　ありますが、それぞれのチャネルは目的ごとに異なります。
　・16ビットタイマの6つのうち5つはシステムが使用するため、現時点でユーザレベルで使用できる
　　のは１つです。
　・システムで予約している16ビットタイマは、以下のような用途で使用されます。

　　1ms割り込み
　　サウンド再生
　　USBクロック供給
　　赤外線送受信

　・8ビットタイマの6つのうち2つはシステムが使用するため、現時点でユーザレベルで使用できる
　　のは4つです。
　・システムで予約している8ビットタイマは、CPUスタンバイ解除のために使用されます。
　・あとひとつはシリアルインターフェースのクロック作成に使用されます。

　・計時タイマは1秒割り込みとして使用されます。

　・全てのタイマのシステムでの使用方法は、将来的に変更される可能性があります。ユーザは現
　　システムでのみ、という条件付でシステムで使用していないタイマを利用することが可能です。

【シリアルインターフェース】
　・S1C33209は内部に独立して動く4ch.のシリアルインターフェースを内蔵しています。
　・P/ECEではハードウェアの仕様上、全てのシリアルインターフェースを利用することができま
　　せん。このため、ハードウェア的に利用できるのは2チャネルのみとなります。このうちの
　　1チャネルはシステムが使用するため、現時点でユーザレベルで使用できるのは１つです。
　　ただし、この1チャネルはP/ECEに外部シリアル入出力端子がないため、ユーザは利用する
　　ことが出来ません。

【A/D変換器】
　・S1C33209は内部に独立して動くA/D変換器を内蔵しています。A/D変換器はアナログ値の計測
　　などを行うことができます。
　・P/ECEではハードウェアの仕様上、全てのA/D変換器を利用することができません。このため、
　　ハードウェア的に利用できるのは2チャネルのみとなります。この2チャネルはシステムが使用
　　するため、現時点でユーザレベルで使用できるA/D変換器はありません。
　・システムで予約しているA/D変換器は、内部電圧監視用に使用されます。

【汎用入出力ポート】
　・S1C33209は内部に多くの入出力ポートを持っています。入出力ポートを使用して外部への信号
　　出力、信号入力、CPUへの割り込み等を行います。

　・S1C33209には汎用入出力ポートが入力ポート13ビット、入出力兼用ポート29ビットあります
　　が、他の内蔵周辺回路との共用部分があるため、全てのポートが使用できるわけではありま
　　せん。

　・全てのポートはシステムが使用するため、現時点でユーザレベルで使用できる入出力ポート
　　はありません。
　・システムで予約している入出力ポートは、以下のような用途で使用されます。

　　クロック制御
　　LCD制御
　　赤外線制御
　　電源制御
　　USB制御
　　ボタン入力

【高速積和演算機能】
　・S1C33209は内部に積和演算機能を内蔵しています。このため、16ビット×16ビット＋64ビット
　　の演算を高速におこなうことができます。また、16ビット×16ビット、及び32ビット×32ビット
　　の乗算を高速におこなうことができます。
【グラフィック機能】 
　グラフィック機能は、APIにより基本的な描画をおこなうことができます。  
　ドット数は128*88、１ピクセルにつき４色の表示能力があります。 
　この他にLCD直接転送APIが使用可能ですが、以下の理由により通常のアプリケーションではできる
　だけ使用しないでください。
　　文字表示・電源表示等、組み込みの描画サービスが使用できません。
　　将来LCD部の仕様が変更された場合に、効果が得られない可能性があります。

【仮想画面バッファ】
　・仮想画面バッファは１ピクセル１バイトで構成されており、下位2ビットのみ使用して４色の表示
　　を行います。
　・仮想画面バッファの上位ビットには必ず０を書き込むようにしてください。０以外が書き込まれて
　　いた場合の動作は保証されません。
　・仮想画面バッファは、画面1枚分の画像データを保存しておき、必要なタイミングでハードウェア
　　にイメージを転送して表示画面を作成します。仮想画面バッファを操作しただけでは画面表示内容
　　に変化はありませんので、注意が必要です。
　・仮想画面バッファの先頭アドレスは必ず4バイト境界に設定してください。提供されている標準の
　　ｃコンパイラ環境では、特に意識する必要はありません。
　・仮想画面バッファの変更を数pceAppProc回に分けて行う場合には、アプリケーション側で仮想画
　　面バッファを複数持ち、pceLCDTransの直前にpceLCDSetBufferで表示を行いたい仮想画面バッ
　　ファを指定するようにしてください。
　・通常想定されている１回のpceAppProcで仮想画面バッファの書き換え、pceAppProcのリターン
　　直前でのpceLCDTrans実行では、特に意識する必要はありません。

【仮想画面バッファの明度】
　　0:白  
　　1:薄灰  
　　2:濃灰  
　　3:黒  

【仮想画面バッファの構成】
　　横128ドット×縦88ドット = 11264バイト  
　　画面左上が仮想画面バッファの先頭アドレス
　　画面右方向に1ピクセル進むにしたがって1バイト
　　画面下方向に1ピクセル進むにしたがって128バイト
